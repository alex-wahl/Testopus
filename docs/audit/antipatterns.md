## Blockers

### AP-01 — Live credentials committed and echoed into logs/reports
- **Location:** `config/yaml_configs/default.yaml:4-5`; `tests/ui_tests/web/gasag/test_gasag.py:50,59`.
- **Why:** Violates the MANDATORY secret-hygiene rule (CLAUDE.md "Safety"). A real-looking
  `username`/`password` is git-tracked (and in history); the tests `print()` the email and password
  field values, which `pytest.ini` `log_cli=1` + CI `-vv` surface into CI logs and the 90-day
  artifacts — and the password sits in the DOM that failure screenshots capture and publish to Pages.
- **Impact:** Irreversible, outward-facing secret exposure — the only finding here you can't undo by
  editing code.
- **Action:** **[devops-engineer + qa-automation-engineer]** Rotate/retire the account now; replace
  with `${ENV_VAR}` interpolation read in `load_config` (`python-dotenv` is already a dep), keep only
  placeholders in `default.yaml`, inject real values via CI secrets; delete the two `print` lines;
  `git rm --cached config/yaml_configs/override.yaml` (see AP-17); ensure no password field is
  screenshotted (see AP-19).

### AP-02 — `@retry` default catches every exception, masking reproducible failures
- **Location:** `core/pom/web/base_page.py:595` (`exceptions=(Exception,)`); applied with the default
  on all 12 methods in `tests/ui_tests/web/gasag/test_gasag.py:38,46,55,64,70,79,128,137,146,165,184,195`.
- **Why:** The default catches `AssertionError`, so a *reproducible* assertion failure is retried 3×
  with a 2s delay before surfacing — exactly what CLAUDE.md forbids ("`@retry` absorbs *genuine* web
  flakiness only"). Separately, `on_retry` is logging-only, so a retry-to-pass emits **no structured
  signal** — destroying the flaky-telemetry the AI/QA roadmap is built on.
- **Impact:** False greens on real bugs (QA-existential); and the highest-value flaky datum is dropped
  on the floor. *Nuance (verified):* on the three soft-assertion tests (`test_wordings` etc.) `@retry`
  is **inert** — `pytest_check` doesn't raise until `pytest_runtest_makereport`, after the function
  returns — so there it is misleading decoration rather than active masking.
- **Action:** **[qa-automation-engineer]** Two distinct fixes: (a) narrow the default `exceptions` to
  web exceptions only (`TimeoutException, StaleElementReferenceException, ElementClickInterceptedException,
  NoSuchElementException, ElementNotInteractableException`) — a one-line change, land immediately; and
  (b) emit a fail-open, side-effect-isolated structured `retry_event` from the decorator (lands with
  the telemetry pipeline — see roadmap). Drop `@retry` from soft-assert-only tests.

## High

### AP-03 — `merge_configs` shallow copy aliases nested dicts
- **Location:** `core/config/config_loader.py:38` (`result = default_config.copy()`).
- **Why:** A shallow copy shares nested dicts by reference. *Nuance (verified):* the recursion rebuilds
  nested dicts that **are** overridden, so only nested dicts present in `default` but **not** overridden
  (and the no-override return path) stay aliased to the session-cached default — narrower than "any
  mutation corrupts the source," but real.
- **Impact:** Latent today (config is read-only) but a landmine the moment `--ai`/`--framework`
  resolution writes back into the config dict (the next decided step — see roadmap), silently
  corrupting the session-scoped default across tests.
- **Action:** **[backend-engineer]** Deep-merge into fresh dicts (`copy.deepcopy` the base, or recurse
  into `{}`); add a Red→Green self-test in `tests/internal_tests/` asserting the source is untouched —
  the canonical TDD target CLAUDE.md names. Fix **before** flag write-back lands.

### AP-04 — CI runs no static / collection gate
- **Location:** `.github/workflows/test.yml` (single `test` job straight to `docker compose run`).
- **Why:** The configured linters/type-checker (`pyproject.toml:83-85`, correct targets) are never run
  in CI, and `pytest --collect-only` — the cheap plugin-compat smoke CLAUDE.md mandates for Dependabot
  PRs — is absent.
- **Impact:** Style/type regressions merge freely; every Dependabot bump (pytest 8→9, pytest-html 3→4,
  pydantic, mypy) lands with no early signal.
- **Action:** **[devops-engineer]** Add a fast non-Docker job (parallel to the test job): `hatch run
  lint && hatch run typecheck && pytest --collect-only`; gate PRs on it.

### AP-05 — Write-scoped third-party Actions pinned to mutable tags
- **Location:** `.github/workflows/test.yml:234` (`EnricoMi/publish-unit-test-result-action@v2`), `:250`
  (`peaceiris/actions-gh-pages@v3`), under `contents: write`.
- **Why:** A moving major tag on a write-scoped third-party action means a hijacked/retagged upstream
  release can push to the `reporting` branch and comment on PRs under `GITHUB_TOKEN`.
- **Impact:** Supply-chain compromise of the repo's published reports.
- **Action:** **[devops-engineer]** SHA-pin both (`@<full-sha>  # vX`); Dependabot already watches
  `github-actions` and will bump the SHAs.

### AP-06 — `test_redirect_to_gasag_portal_after_login` is mislabeled coverage
- **Location:** `tests/ui_tests/web/gasag/test_gasag.py:65-68` (fixture builds + loads the login URL at
  `:31-36`; `BasePage.__init__` calls `driver.get(url)` at `core/pom/web/base_page.py:24-26`).
- **Why:** The test never calls `login()`. It asserts `wait_for_url_change(login_url)` against the very
  URL it was just told to load — so it passes only on *incidental* URL mutation and fails on a stable
  login page, while its name claims it verifies post-login redirect.
- **Impact:** Worse than a no-op — it's an active lie about coverage; a human (and future flaky-triage)
  reads green here as "login redirect works."
- **Action:** **[qa-automation-engineer]** Rewrite to `login()` then assert the portal URL via
  `wait_for_url_contains(<portal-path>)`, or delete/rename it.

### AP-07 — `js_input` builds JavaScript via f-string interpolation
- **Location:** `core/pom/web/base_page.py:248` (`execute_script(f"arguments[0].value = '{text}';", element)`).
- **Why:** Any `'`, `\`, newline, or `</script>` in `text` corrupts/injects the script; it's the JS
  fallback inside `safe_input`, so it fires precisely on the hard cases (e.g. a password containing a
  quote).
- **Impact:** Flaky/incorrect input on legitimate data; latent injection seam (more dangerous once AI
  generates inputs).
- **Action:** **[qa-automation-engineer / backend-engineer]** Pass the value as a bound argument:
  `execute_script("arguments[0].value = arguments[1];", element, text)`.

### AP-08 — Driver hardcoded; no `core/drivers/` seam; `BasePage` bound to Selenium
- **Location:** `fixtures/setup.py:40,42` (`webdriver.Chrome(...)` inline), `core/pom/web/base_page.py:24`
  (`__init__(self, driver: WebDriver, ...)`); `core/drivers/` does not exist.
- **Why:** Violates DIP/OCP — the only extension point for "which framework runs" is editing the
  fixture, and every page object is cemented to Selenium. Blocks the multi-framework `--framework` flag
  and makes AI layers un-mockable.
- **Impact:** The #1 *structural* constraint; blocks Playwright/Appium and the AI roadmap.
- **Action:** **[backend-engineer + architect]** Build the `core/drivers/` Protocol seam (already decided
  as Rec #2 of the prior council doc) — see [roadmap.md](roadmap.md). Listed here as the standing
  architectural antipattern; the fix is sequenced in the roadmap, not duplicated.

## Medium

### AP-09 — Allure marker→label pipeline is dead code
- **Location:** `fixtures/allure.py:209-218` maps `feature/story/severity/tag`; `pytest.ini:11-13` has
  all markers commented out and no test uses `@pytest.mark.*` (the docstring→description branch at
  `allure.py:204-206` *does* fire).
- **Why/Impact:** The published GitHub Pages trend report — the framework's headline output — has no
  feature/story/severity grouping; everything lands flat.
- **Action:** **[qa-automation-engineer]** Register the four markers in `pytest.ini` (+ `--strict-markers`),
  decorate the gasag suite, and add a marker to the `testopus-web-test` skill template.

### AP-10 — Hand-rolled polling loops reimplement `WebDriverWait`
- **Location:** `core/pom/web/base_page.py:87-98,187-207,267-298` (`while time.time()-start < timeout: … time.sleep(0.5)`).
- **Why/Impact:** `WebDriverWait(driver, timeout, poll_frequency=0.5, ignored_exceptions=…)` already
  polls and swallows transient exceptions; the loops are duplicated control flow, and an inner
  `timeout=1` masks the caller's budget (DRY/KISS, against the "explicit waits over sleeps" convention).
- **Action:** **[qa-automation-engineer]** Replace with `WebDriverWait(..., ignored_exceptions=(StaleElementReferenceException,)).until(...)`; keep the click/JS-click fallback driven off one wait.

### AP-11 — `--framework` / `--ai` / `--config` parsed but never read
- **Location:** `fixtures/cli.py:27-40`; only `override` is consumed (`core/config/config_loader.py:71`).
- **Why/Impact:** A CLI surface that silently no-ops. `--config` is especially misleading (its help says
  "Required") yet `load_config_from_cli` ignores it and always loads the hardcoded `default.yaml`.
- **Action:** **[backend-engineer]** Until wired, fail loudly on unsupported `--framework`/`--ai` values
  (`pytest.UsageError`) and thread `--config` into `load_config_from_cli`; full wiring is in the roadmap.

### AP-12 — Dead, invalid, duplicated locator constants
- **Location:** `core/pom/web/gasag/login_page.py:47-49`.
- **Why/Impact:** `APPLE_LOGIN_TEXT = (By.CLASS_NAME, "social__legal-section__show d-hide")` is a
  **compound** class — invalid for `By.CLASS_NAME` (raises `InvalidSelectorException` if used) and tied
  to a `d-hide` visibility-state class; `GOOGLE_LOGIN_TEXT` and `APPLE_LOGIN_TEXT_2` are byte-identical
  duplicates; all three locator constants are unused (tests use inline JS instead — see AP-13).
- **Action:** **[qa-automation-engineer / frontend-engineer]** Delete the dead trio or replace with one
  valid scoped `By.CSS_SELECTOR` without the state class.

### AP-13 — Inline selectors / raw-driver JS inside tests
- **Location:** `tests/ui_tests/web/gasag/test_gasag.py:152-156,171-175` (`document.querySelector('button.google-btn.social-btn')`, `.nextElementSibling.querySelector('label')`).
- **Why/Impact:** Directly violates "locators live only as page-class constant tuples — never inline a
  selector in a test." Brittle sibling-walk CSS; invisible to future locator self-healing (which reads
  page objects).
- **Action:** **[qa-automation-engineer]** Move to `LoginPage` locator constants + an action method;
  delete the raw-JS path.

### AP-14 — Docker base image and browser/driver unpinned
- **Location:** `docker/Dockerfile:1` (`python:3.12-slim`, floating), `:18-19` (`chromium`,
  `chromium-driver` via bare `apt-get`).
- **Why/Impact:** Every rebuild can pull a different Chromium/driver — the axis most likely to change
  UI-test behavior; "Docker mirrors CI" holds only until the next Debian bump.
- **Action:** **[devops-engineer]** Digest-pin the base image, version-pin the Chromium/driver apt pair,
  record the resolved Chromium version into `environment.properties`. (Note the deliberate tension with
  CLAUDE.md's "no `==` pins" ethos — record the exception in an ADR; apt packages aren't Dependabot-managed.)

### AP-15 — Docker buildx cache key never hits
- **Location:** `.github/workflows/test.yml:90` (`key: …-buildx-${{ github.sha }}`).
- **Why/Impact:** A per-commit SHA never matches on a fresh commit; only `restore-keys` salvages a
  partial restore while `cache-to` writes a fresh entry every run (cache churn, slow builds).
- **Action:** **[devops-engineer]** Key on `hashFiles('docker/Dockerfile','pyproject.toml')`.

### AP-16 — Unused heavy dependencies in core `dependencies`
- **Location:** `pyproject.toml:36-46` — `Appium-Python-Client`, `playwright`, `responses`, `Faker`,
  `requests` are imported nowhere (grep-confirmed); `pydantic` was unused **at audit time** but is
  now actively used for config validation (`core/config/schema.py`) — no longer a dead dep.
- **Why/Impact:** Install/build weight + attack surface + Dependabot churn for packages nothing calls —
  the same antipattern already cleaned up for `langchain`/`pydantic-ai`/`pyviztest`, and against
  CLAUDE.md's "add a dependency only when code imports it."
- **Action:** **[devops-engineer / backend-engineer]** Move `Appium`/`playwright` to a `[frameworks]`
  extra, `requests`/`Faker` to `[test]`, delete `responses`; keep `pydantic` with an inline note.

### AP-17 — `override.yaml` is git-tracked and schema-mismatched
- **Location:** `config/yaml_configs/override.yaml` (tracked per `git ls-files`); its keys don't exist in
  `default.yaml`, so `merge_configs` just appends junk.
- **Why/Impact:** The prior council doc said "gitignore `override.yaml`" but it's already committed —
  gitignoring won't untrack it.
- **Action:** **[devops-engineer / backend-engineer]** `git rm --cached config/yaml_configs/override.yaml`,
  add to `.gitignore`, and fix or remove the stale keys.

### AP-18 — No commit-SHA / run-id provenance in telemetry
- **Location:** `fixtures/allure.py:179-187` writes branch + timestamp; no `GITHUB_SHA`/`GITHUB_RUN_ID`.
- **Why/Impact:** Rows can't be joined to a commit or CI run — the decided telemetry pipeline is keyed on
  `run_id` for idempotency, so this is its missing prerequisite; trend regressions can't be bisected.
- **Action:** **[data-engineer / devops-engineer]** Add `Commit`, `RunId`, `RunAttempt`, `CiUrl`
  (env-driven, fail-open) to `environment.properties`.

### AP-19 — PII (password DOM) rendered into failure screenshots
- **Location:** `fixtures/allure.py:115-123` (`save_screenshot` + attach), after the login flow has typed
  the password into the form.
- **Why/Impact:** Credentials/PII rendered into a screenshot uploaded as a CI artifact and published to
  public Pages.
- **Action:** **[qa-automation-engineer / data-engineer]** Blank/mask sensitive fields before screenshot
  + attach (ties to the prior doc's `redact()` work).

### AP-20 — Timestamp regex rewrites the published report JSON
- **Location:** `ci/scripts/customize_allure_report.py:605-632` (`fix_json_timestamps`).
- **Why/Impact:** *Nuance (verified):* it runs on the **rendered** `allure-report` dir, not the canonical
  `allure-results` (which is uploaded untouched) — so source data is safe. But it breaks ISO-8601 in the
  published JSON via a sloppy `(\d{4}).(\d{2}).(\d{2})` pattern (the `.` matches any char), corrupting
  any consumer of the published report's JSON.
- **Action:** **[devops-engineer / data-engineer]** Establish the hard rule "extract telemetry from raw
  `allure-results`, never the rendered report"; scope/curb the regex (exclude `widgets/`/`history*`).
