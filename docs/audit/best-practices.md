Industry and framework best practices that fit Testopus, each with the **current gap** (`file:line` where
relevant), **how to adopt** it, and a **reference** to the official source (verified docs-first). These are
recommendations, not yet applied.

## Testing & Pytest

- **Explicit waits via `WebDriverWait(poll_frequency, ignored_exceptions)`, not sleep loops.** *Gap:*
  `core/pom/web/base_page.py:87-98,187-207,267-298` hand-roll `time.sleep` polling. *Adopt:* one
  `WebDriverWait(..., ignored_exceptions=(StaleElementReferenceException,)).until(...)` per wait; Selenium
  polls at 0.5s by default. *Ref:* <https://www.selenium.dev/documentation/webdriver/waits/>
- **Parameterize `execute_script` data; never f-string it.** *Gap:* `core/pom/web/base_page.py:248`;
  `tests/ui_tests/web/gasag/test_gasag.py:152-156,171-175`. *Adopt:* pass values via `arguments[1..]`.
  *Ref:* <https://www.selenium.dev/documentation/webdriver/interactions/>
- **Register every marker; run `--strict-markers`.** *Gap:* `pytest.ini:11-13` (commented). *Adopt:*
  populate `[pytest] markers`, add `--strict-markers` so an unregistered marker fails rather than warns.
  *Ref:* <https://docs.pytest.org/en/stable/how-to/mark.html>
- **`@retry` for genuine web flakiness only; never `AssertionError`; never on soft-assert tests.** *Gap:*
  `core/pom/web/base_page.py:595` defaults to `(Exception,)`. *Adopt:* default to web exceptions; drop
  `@retry` from `pytest_check` tests (it can't observe soft assertions). *Ref:* CLAUDE.md "Conventions" +
  <https://github.com/okken/pytest-check>
- **Assert stable substrings/states, not full localized sentences.** *Gap:*
  `core/pom/web/gasag/login_page.py:13-32` + the 9-way copy block in `test_gasag.py:82-126` assert long
  German strings. *Adopt:* assert a stable substring or an error code/state; move copy checks to a separate
  content tier. *Ref:* <https://docs.pytest.org/en/stable/how-to/assert.html>
- **TDD the pure framework logic.** *Gap:* `tests/internal_tests/test_framework.py` is one trivial assert.
  *Adopt:* write failing self-tests for `merge_configs`, `utils/helpers`, and the retry/wait contract first.
  *Ref:* CLAUDE.md "Development methodology — TDD".

## Locators & frontend

- **Locator-priority ladder.** data-testid → ARIA role/name → stable id/name → scoped CSS → text; never
  positional XPath or compound state-class chains. *Gap:* no ladder documented; `login_page.py:48` is a
  concrete violation. *Adopt:* add the ladder to CLAUDE.md "Conventions" and the `testopus-page-object`
  skill. *Ref:* <https://www.selenium.dev/documentation/test_practices/encouraged/locators/> ·
  <https://playwright.dev/python/docs/locators>
- **Request `data-testid` hooks from the app team.** *Gap:* the suite leans on framework-generated IDs
  (`signInName`, `dummyGoogleBtn`) that may change on platform upgrades. *Adopt:* file a request for stable
  test ids on key controls. *Ref:* <https://playwright.dev/python/docs/locators#locate-by-test-id>

## CI / DevOps

- **Always run `pytest --collect-only` + lint/type in CI** (especially for Dependabot PRs). *Gap:*
  `.github/workflows/test.yml` runs neither. *Adopt:* a fast static job parallel to the test job. *Ref:*
  CLAUDE.md "Dependency currency" + <https://docs.pytest.org/en/stable/how-to/usage.html>
- **SHA-pin third-party Actions.** *Gap:* `test.yml:234,250` on mutable tags under `contents: write`.
  *Adopt:* `@<full-sha>  # vX`; let Dependabot bump. *Ref:*
  <https://docs.github.com/en/actions/security-for-github-actions/security-guides/security-hardening-for-github-actions#using-third-party-actions>
- **Never commit secrets; inject via env / CI secrets.** *Gap:* `config/yaml_configs/default.yaml:4-5`.
  *Adopt:* `${VAR}` placeholders in YAML, `os.environ`/`load_dotenv` in the loader, repo secrets in CI.
  *Ref:* <https://docs.github.com/en/actions/security-for-github-actions/security-guides/using-secrets-in-github-actions>
- **Pin the base image by digest + reproducible apt installs.** *Gap:* `docker/Dockerfile:1,18-19`.
  *Adopt:* `FROM python:3.12-slim@sha256:…`; install `chromium=<ver> chromium-driver=<ver>`; record an ADR
  for the deliberate pin exception. *Ref:* <https://docs.docker.com/build/building/best-practices/>
- **Stable build-cache keys.** *Gap:* `test.yml:90` keys on `github.sha`. *Adopt:*
  `hashFiles('docker/Dockerfile','pyproject.toml')`. *Ref:*
  <https://docs.github.com/en/actions/using-workflows/caching-dependencies-to-speed-up-workflows>
- **Run CI containers as a non-root host UID.** *Gap:* `test.yml:71` `chmod -R 777`. *Adopt:* `--user
  "${UID}:${GID}"`, drop the blanket chmod. *Ref:* <https://docs.docker.com/engine/security/>

## Architecture

- **Driver-as-interface (Protocol + factory) so the POM is framework-agnostic.** *Gap:* `fixtures/setup.py`
  + `base_page.py:24` bind to Selenium. *Adopt:* `BaseDriver`/`ElementHandle` Protocols + `create_driver()`
  factory in `core/drivers/`; the fixture and `BasePage` depend on the Protocol. *Ref:*
  <https://www.selenium.dev/documentation/webdriver/> · <https://playwright.dev/python/docs/intro>
- **ADRs for structural decisions.** *Gap:* `.claude/adr/` doesn't exist. *Adopt:* a MADR-style
  `.claude/adr/NNNN-title.md` per boundary change (context / decision / rejected alternative). *Ref:*
  <https://adr.github.io/>
- **Optional layers via packaging extras; AI/ML/data depend on core, never the reverse.** *Gap:*
  `pyproject.toml:36-46` bundles Appium/Playwright in core deps. *Adopt:* named extras; an import-guard
  self-test enforces the one-way dependency. *Ref:*
  <https://packaging.python.org/en/latest/specifications/dependency-specifiers/>
- **One import-resolution mechanism.** *Gap:* `sys.path.insert` in `fixtures/cli.py:7` +
  `core/config/config_loader.py:7` vs `pytest.ini` `pythonpath=.`. *Adopt:* drop the `sys.path` hacks.
  *Ref:* <https://hatch.pypa.io/latest/config/build/>
- **Fail loud on unsupported flags.** *Gap:* `fixtures/cli.py:27-40` accepts `--framework playwright` and
  silently runs Selenium. *Adopt:* validate in `pytest_configure`, raise `pytest.UsageError` until wired.
  *Ref:* <https://docs.pytest.org/en/stable/reference/reference.html#pytest.hookspec.pytest_addoption>

## Config & backend

- **Deep merge with no source aliasing + fail-fast typed validation.** *Gap:* `core/config/config_loader.py:38`
  shallow-copies; `:20,64` has no load error handling. *Adopt:* `copy.deepcopy` the base; validate the parsed
  result (Pydantic v2 is already a dependency) and raise a keyed `ConfigError`. *Ref:*
  <https://docs.pydantic.dev/latest/>

## AI / ML / data

- **API key via env only; fail-open on API errors; tier models cheapest-first.** *Adopt:* instantiate the
  `anthropic` client with no key arg (reads `ANTHROPIC_API_KEY`); wrap calls in try/except and fall back to
  "no suggestion"; default bulk work to Haiku 4.5, escalate to Sonnet 4.6, reserve Opus 4.8. *Ref:*
  <https://docs.claude.com/en/api/client-sdks> · <https://docs.claude.com/en/docs/about-claude/models/overview>
- **Budget tokens: `count_tokens`, prompt caching, the Batch API.** *Adopt:* size payloads before large
  screenshot/DOM sends; cache the static system prompt; use message batches (50% discount) for history
  backfill. *Ref:* <https://docs.claude.com/en/docs/build-with-claude/token-counting>
- **AI advises; the test owns the verdict.** *Adopt:* the only runtime AI/ML touchpoint is *emission*
  (fail-open, side-effect-isolated `retry_event` / capture) — never control flow; self-healing is
  propose-only, eval-gated, and out of both the red and green paths. *Ref:* CLAUDE.md "Toward an AI-Driven QA
  Framework" + `.claude/skills/flaky-triage`.
- **Extract ML features from raw `allure-results`, never the rendered report; ISO-8601 in data, format at
  the edge.** *Gap:* `ci/scripts/customize_allure_report.py:605-632` rewrites timestamps in the published
  JSON. *Adopt:* read `reports/allure-results/*-result.json`; keep timestamps ISO-8601; format only in HTML.
  *Ref:* <https://allurereport.org/docs/how-it-works-test-result-file/>
- **No model without a labeled eval set + baseline; state the asymmetric cost.** *Adopt:* a transparent
  rules-based flaky score is the baseline any classifier must beat on a frozen labeled set; fix a max
  false-quarantine rate (a quarantined real bug ≫ a tolerated flake); perceptual diff (SSIM + masks) before
  any learned visual model. *Ref:* <https://scikit-learn.org/stable/modules/model_evaluation.html> ·
  <https://scikit-image.org/docs/stable/auto_examples/transform/plot_ssim.html>
- **One pipeline feeds ml and ai (DRY); fail-open, idempotent ETL off the hot path; provenance + PII
  redaction on every row.** *Gap:* no extraction layer; `fixtures/allure.py:179-187` lacks commit/run-id;
  screenshots carry PII. *Adopt:* a single `core/telemetry/` reader emitting one normalized record with
  `{commit, run_id}` provenance and redaction before persist. *Ref:*
  <https://allurereport.org/docs/how-it-works-test-result-file/>
