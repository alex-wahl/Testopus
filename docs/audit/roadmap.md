**How to read this.** The *forward* architecture — driver seam, telemetry pipeline, flaky-triage as the
first AI feature, the ml/ai split, ADRs — was already decided by the council in
[`.claude/council/ai-driven-qa-and-claude-md-adoption.md`](../../.claude/council/ai-driven-qa-and-claude-md-adoption.md).

- **[DECIDED]** — execute the prior council decision; the cited recommendation is the source of truth.
- **[NEW]** — surfaced by *this* audit (mostly the correctness / CI / hygiene layer the prior council
  didn't open) and not covered there.

Each item lists **Affects** (what changes) and **Value** (why it's worth doing). Owners are role labels.

**Implementation status** is marked per item — **✅ done** · **⚠️ partial / manual step pending** ·
**⬜ pending (roadmap)** — and consolidated in [STATUS.md](STATUS.md). Done items landed on branch
`fix/antipatterns-audit` (verify with `git diff main...fix/antipatterns-audit`). Remaining work is the
AI/ML layer; code changes still go through the `code-reviewer` → `writer` gate.

> **Historical note:** This roadmap was written against the gasag suite. The gasag suite was later
> replaced by the public Toolshop demo target and an API layer was added. See ADR-0003.

## Now — correctness, security, and signal (small, high-leverage, mostly new)

These are cheap, mostly pure-logic or config changes that remove real risk and unblock everything later.
**Status: ✅ all done** (one manual residual — credential rotation).

- **✅ [NEW] Secret remediation.** Rotate the committed credential, move to `${ENV_VAR}` interpolation,
  inject via CI secrets, drop the credential `print`s, untrack `override.yaml`.
  *Affects:* `config/yaml_configs/default.yaml`, `core/config/config_loader.py`, `.github/workflows/test.yml`,
  `tests/ui_tests/web/gasag/test_gasag.py:50,59`. *Value:* closes the only irreversible/security
  exposure; makes the repo safe to fork/open-source. (owner: devops-engineer + qa-automation-engineer) — fixes AP-01, AP-17.
  **⚠️ Manual residual:** rotate/retire the live GASAG credential (cannot be done in code).

- **✅ [NEW] CI quality gate.** Add a fast non-Docker job: `hatch run lint && hatch run typecheck && pytest
  --collect-only`, parallel to the test job.
  *Affects:* `.github/workflows/test.yml`. *Value:* enforces the already-configured linters/types and gives
  every PR and Dependabot bump a cheap plugin-compat signal. (owner: devops-engineer) — fixes AP-04.

- **✅ [NEW] SHA-pin write-scoped third-party Actions.**
  *Affects:* `.github/workflows/test.yml:234,250`. *Value:* closes a supply-chain hole on the actions that
  write the published report. (owner: devops-engineer) — fixes AP-05.

- **✅ [DECIDED + NEW] `@retry` default narrowing + `retry_event` emission.** Narrow the default `exceptions`

  *Affects:* `core/pom/web/base_page.py:595`. *Value:* stops masking reproducible failures **and** unlocks
  the flaky-to-pass signal the entire AI/QA roadmap depends on. (owner: qa-automation-engineer) — fixes AP-02.

- **✅ [NEW] `merge_configs` deep-merge fix + first real `internal_tests`.**
  *Affects:* `core/config/config_loader.py:38`, `tests/internal_tests/`. *Value:* removes a config-corruption
  landmine **before** flag write-back lands; satisfies the TDD mandate on the module CLAUDE.md singles out.
  (owner: backend-engineer) — fixes AP-03, AP-23.

- **✅ [NEW] Activate the Allure marker taxonomy.** Register `feature/story/severity/tag` (+ `--strict-markers`),

  *Affects:* `pytest.ini`, `tests/ui_tests/web/gasag/test_gasag.py`, `.claude/skills/testopus-web-test`.

  Allure keying. (owner: qa-automation-engineer) — fixes AP-09.

- **✅ [NEW] Quick correctness sweep.** Parameterize `js_input` (AP-07); rewrite or delete the mislabeled
  redirect test (AP-06); lift inline-JS selectors into the page object + delete the dead/invalid locators
  (AP-12, AP-13).
  *Affects:* `core/pom/web/base_page.py:248`, `tests/ui_tests/web/gasag/test_gasag.py`,
  `core/pom/web/gasag/login_page.py`. *Value:* removes real input/locator bugs and a green-for-wrong-reason

## Next — the structural seams (decided architecture + the hygiene to support it)

**Status: ✅ done except the telemetry pipeline (⬜) and flaky-triage (⬜).**

- **✅ [DECIDED] Driver-factory seam + ADR-0001.** `core/drivers/{base,factory,selenium}.py` — `BaseDriver`/
  `ElementHandle` as `@runtime_checkable Protocol`, `execute_script` a Selenium-only escape hatch.
  *Scope (KISS, confirmed in cross-critique):* abstract only the ~7 driver verbs + ~6 element verbs the
  current `LoginPage`/tests use, **not** all ~50 `BasePage` helpers (which stay as composition on top of
  the Protocol). The gasag suite passing unchanged is the regression gate.
  *Affects:* `fixtures/setup.py`, `core/pom/web/base_page.py:24`, new `core/drivers/`, `.claude/adr/0001-*.md`.
  *Value:* the #1 structural unlock — Playwright/Appium slot in without forking page objects, and AI layers
  become mockable. (owner: backend-engineer + architect) — prior doc Rec #2; fixes AP-08.
  **⚠️ Gate:** the live gasag UI regression (`hatch run ui:web`, real browser + site) must pass in CI.

- **⬜ [DECIDED] One telemetry pipeline.** `core/telemetry/{models,extract}.py` — Pydantic v2 models
  (`schema_version`, NDJSON) for run/result/trend_point; idempotent on `run_id`, fail-open, off the hot
  path; **read raw `allure-results`, never the rendered report.**
  *Affects:* new `core/telemetry/`, plus the NEW prerequisites: commit/run-id provenance (AP-18 ✅) and the
  extract-from-raw rule (AP-20 ✅). *Value:* turns the inert 90-day history into a queryable dataset feeding
  both ml and ai from one source (DRY). (owner: backend-engineer + data-engineer) — prior doc Rec #3.
  **Prerequisites already done; the pipeline itself is pending.**

- **✅ [DECIDED] Wire `--ai` / `--framework` / `--config`.** `--framework` now resolves through the driver
  factory (unsupported values raise `pytest.UsageError`); `--config` is threaded into `load_config_from_cli`.
  *Affects:* `fixtures/cli.py`, `core/config/config_loader.py`. *Value:* removes the misleading dead flags.
  (owner: backend-engineer + qa-automation-engineer) — prior doc Now-action; fixes AP-11.
  **⬜ `--ai` stays reserved/inert** until the first AI feature gives it something to gate.

- **✅ [NEW] Dependency & docs honesty pass.** Move framework deps to extras; prune the `api`/`mobile` script
  references; align the `pyproject.toml` description with README's honest framing; refresh the stale `src/`
  wording.
  *Affects:* `pyproject.toml`, `.github/workflows/test.yml`, `README.md`, `CLAUDE.md`. *Value:* a
  deterministic core install (KISS) and docs the AI agents can trust. (owner: devops-engineer + writer) —
  fixes AP-16, AP-25.

- **✅ [DECIDED] `LOCATOR_PRIORITY` ladder.** data-testid → ARIA role/name → stable id/name → scoped CSS →
  text; never positional XPath / compound state-class.
  *Affects:* `.claude/skills/testopus-page-object`, CLAUDE.md "Conventions". *Value:* reviewers and the
  page-object skill enforce stable hooks. (owner: frontend-engineer + qa-automation-engineer) — prior doc Rec #1.

- **✅ [DECIDED] `redact()` before Allure attach + screenshot scrub.**
  *Affects:* `fixtures/allure.py`, `utils/redact.py`. *Value:* no secret/PII reaches a published report. (owner:
  devops-engineer + qa-automation-engineer) — prior doc Rec #6; fixes AP-19.

- **⬜ [DECIDED] flaky-triage v1 (read-only over telemetry).** First AI feature: classify failures
  (locator-stale / timing-flaky / app-bug / env) from telemetry + redacted screenshots; never touches the
  verdict.
  *Affects:* `.claude/skills/flaky-triage`, an offline `core/ai/` consumer. *Value:* turns the signal we
  already emit into root-cause classification at the cheapest model tier. (owner: ai-engineer) — prior doc Now/Next.
  **Gated on the telemetry pipeline.**

- **⚠️ [NEW] Docker reproducibility.** Digest-pin base image, version-pin Chromium/driver, fix the cache key;
  run the container as host UID.
  *Affects:* `docker/Dockerfile`, `.github/workflows/test.yml`. *Value:* reproducible Selenium runs;
  comparable trend history. (owner: devops-engineer) — fixes AP-14, AP-15, AP-26.
  **⚠️ Partial:** base image digest-pinned, cache key + host-UID done; exact Chromium apt-version pin
  deferred (brittle without a `snapshot.debian.org` source).

## Later — learning layers (decided, gated) and maintainability

**Status: ⬜ all pending — intentionally deferred (gated on telemetry + a labeled eval set).**

- **⬜ [DECIDED] ml: failure clustering + interpretable flaky score.** TF-IDF/cosine over failure
  messages/traces (unsupervised, no labels); a transparent rules-based flaky score as the **baseline** any
  model must beat. *Value:* collapse many reds into one root cause; honest, actionable signal. (owner:
  ml-engineer) — prior doc Rec #4.

- **⬜ [DECIDED] ai: LLM cluster labeling + `ai-eval` precision gate; locator self-healing; visual diff.**
  Self-healing stays **propose-only**, eval-gated, and out of *both* the red and green runtime paths; SSIM/
  pixel diff with masks. *Value:* higher-order AI QA without ever owning the verdict. (owner: ai-engineer +
  frontend-engineer) — prior doc Later-actions.

- **⬜ [DECIDED] Supervised flaky classifier — gated.** Only if it beats the rules baseline on a frozen eval
  set; ≥50–100 labels across ≥2 apps. *Value:* marginal accuracy over the baseline, *iff* the data proves
  it. (owner: ml-engineer) — prior doc Later-action.

- **⬜ [NEW] Refactor `customize_allure_report.py` (944 lines) into a tested package.** Split branch/dates/
  history/assets behind the current CLI; delete the runtime "create template if missing" cruft; unit-test
  the pure string transforms. *Affects:* `ci/scripts/`, `tests/internal_tests/`. *Value:* maintainability of
  the publishing pipeline. *Do when the Allure flow next needs a change — not speculatively.* (owner:
  devops-engineer)

- **⚠️ [NEW] Decide `tests/api_tests/`: prune now, build when on-roadmap.** ✅ The dangling references were
  pruned (the `api` Hatch script + `workflow_dispatch` option are gone); ⬜ building a real API-test
  foundation is deferred until a need lands. (owner: architect + backend-engineer) — see AP-25.

## Sequencing rationale

1. **Now** items are cheap and remove irreversible risk (secrets), false verdicts (`@retry`, redirect test),
   and a latent corruption (`merge_configs`) — plus the CI gate that protects everything after. **(done)**
2. **Next** is gated on the **driver seam** and **telemetry pipeline**, the two seams every later feature
   depends on; the deepcopy fix must precede flag write-back. **(seam done; telemetry pipeline pending)**
3. **Later** is the learning layer — explicitly gated on a labeled eval set and a baseline, and on AI never
   entering the verdict path. **(pending)**
